Data & Model Documentation

How this simulator was built — the bioreactor runs, statistical signals, surrogate model, and Bayesian optimisation framework underlying the predictions.

Dataset overview

The simulator is grounded in data from 9 successful bioreactor runs conducted at Nurexone using human bone marrow-derived MSCs expanded on FibraCel microcarriers in a 1.5–5.5L stirred tank bioreactor. Four additional runs were excluded due to equipment failure (headplate leaks: RD1031, RD1047), contamination (RD1037, RD1039), or protocol deviation (RD1042a). Each run was 8–17 days total, with minute-by-minute online sensor data (DO, pH, agitation, temperature, gas flows) and 20–30 offline measurements per run.

Statistical signals

Spearman rank correlations were computed between each process variable and log₁₀(total EVs) across the 6 standardised-protocol runs (RD1041, RD1042b, RD1049, RD1052, RD1043, RD1053). Only one predictor reaches statistical significance at α=0.05.

Important confound: Agitation and antibiotics are correlated in this dataset — all antibiotic runs also used 300 rpm agitation. Their individual effects cannot be cleanly separated until a run is designed to vary one while holding the other fixed. The simulator treats them as independent, which introduces uncertainty.

Secretion kinetics

Four runs have time-resolved EV concentration data measured at 9–16h intervals throughout the collection phase. This reveals the shape of the secretion curve — critical for deciding when to harvest.

~110h
secretion peak (RD1041: 111h, RD1042b: 112h, RD1053: 103h)
70h
RD1040 peak — likely truncated by early harvest
~4.6 d
optimal collect duration implied by kinetics data

The kinetics shape is consistent across runs: a slow initial rise (0–24h), rapid accumulation (24–72h), a plateau approaching maximum (72–112h), then saturation. Runs harvested at 3d (72h) capture only ~70% of the secretion curve. This explains why collection days is the strongest predictor of yield — it is essentially a proxy for how much of the secretion curve is captured.

Key purity finding: DNA contamination at crude is consistent regardless of upstream conditions (~2×10¹⁰ EVs/µg DNA across all runs). It is a fixed biological parameter of MSC-conditioned medium — not something upstream optimisation can improve. Protein at crude shows more variation (CV ~21%), and may correlate with secretion phase length and cell density. The TFF diafiltration step is the primary driver of protein removal — filtration steps alone (Agent V, 5µm, 0.45µm) have minimal effect on protein concentration.

Surrogate model

The simulator uses a Gaussian Process (GP) surrogate model fitted to the 6 standardised-protocol runs (RD1030 and RD1033 were excluded as they used a different protocol regime with 30g FibraCel and perfusion). The GP was implemented using a Matérn 5/2 kernel with automatic relevance determination (ARD), fitted via maximum marginal likelihood with 15 random restarts.

# Prediction model — key coefficients (simplified for display)
EVs = 4.6
  + (5.0 − expand_days) × 0.23   # shorter expansion → higher yield
  + collect_contribution(days)   # +0.40/d up to 5.5d, then +0.07/d
  + agit_effect(rpm)   # +0.58 at 225 vs 300 rpm
  − 0.58 × antibiotics
  − 0.65 × compact_fibracell
Honest uncertainty statement: Leave-one-out cross-validation gives a prediction error of ±51% (0.18 log₁₀ units). With n=6 training points and correlated variables, the GP reliably identifies the direction of improvement but not exact yield values. Treat all predictions as directional guidance — not engineering specifications. The confidence score in the simulator reflects proximity to observed data, not absolute accuracy.

Bayesian optimisation framework

The GP surrogate enables Bayesian Optimisation (BO) — a principled method for finding optimal process conditions when each experiment is expensive. Rather than grid-searching, BO uses the GP's uncertainty estimates to compute an Expected Improvement acquisition function that balances exploitation (try near the best known point) and exploration (try where uncertainty is high).

1
Phase 1 — Replicate (next 1–2 runs)
Run RD1041 conditions exactly: 5.0d expansion, 5.0d collection, 225 rpm, no antibiotics, loose FibraCel packing. If yield ≥5.0×10¹³ → signal is confirmed and biological, not a one-off. If not → donor variability is dominating and more replicates are needed before optimising further.
2
Phase 2 — Disentangle confounds (runs 3–5)
Design one run varying agitation alone (e.g. 250 rpm with all else fixed at RD1041 conditions) and one run with antibiotics=YES at 225 rpm. This breaks the agitation–antibiotics confound and gives the GP real contrast to resolve each variable's effect independently.
3
Phase 3 — Explore beyond (runs 6–10)
By run 8–10 the GP has enough contrast to recommend genuinely novel conditions — for example, testing 6d collection, or the collect/expansion interaction. This phase also generates the labeled dataset (run conditions → yield) needed to eventually scaffold an MPC or DRL-based within-run control system.

How to update this simulator

As new runs are completed, open the HTML file in GitHub's editor (press . in your repository) and update two sections in the JavaScript:

1. Add new run to RUNS array — find the var RUNS=[ block and add a new line with the run's parameters and measured yield.

2. Adjust predict() coefficients — find the function predict( block and update the multiplier values to better fit the growing dataset. The comments in that function label each coefficient.

3. Update REF object — if a new run becomes the best observed, update var REF= so all delta calculations compare against the new benchmark.
